# 01. Установка и запуск



## Назначение проекта



MineTracker — Telegram-бот для автоматизированного мониторинга серверов Minecraft. Бот позволяет пользователям отслеживать онлайн, пинг, MOTD и TPS любимых серверов напрямую в Telegram. Основной функционал бота включает персональные подписки на мониторинг, добавление статуса серверов в супергруппы (и их темы/Threads), а также премиум-систему с разделением ролей (юзер, модератор, админ).



Часть данных кэшируется (команды, спам-фильтры) с помощью Redis для предотвращения Rate Limit-ов со стороны Telegram API, а сами данные о серверах и чатах хранятся в PostgreSQL.



---



## Требования к окружению



| Инструмент | Версия |

|---|---|

| Python | 3.13 (рекомендуется) |

| uv / pip | для управления зависимостями |

| PostgreSQL | 15.x или новее |

| Redis | 7.x или новее |



Вы можете запустить базу данных и кэш локально или использовать поставляемый `docker-compose.yml` для быстрого развертывания инфраструктуры.

## Переменные окружения



Проект использует несколько ключевых переменных окружения для работы. Они могут быть переданы через систему, либо загружены из файла `.env` в корне проекта (через Pydantic Settings).



Вы можете скопировать `config.yaml.example` / `.env.example` (если есть) или создать свой `.env` файл со следующим содержимым:



### `BOT_TOKEN` (Обязательно)



**Назначение**: Токен вашего Telegram-бота, полученный от @BotFather.



**Пример**:

```env

BOT_TOKEN=123456789:ABCDefghIJKLmnopQRSTuvwxYZ

```



### `DB_DSN`



**Назначение**: Асинхронный URL для подключения к базе данных PostgreSQL через драйвер `asyncpg`. Если не задан, используется значение по умолчанию.



**Пример**:

```env

DB_DSN=postgresql+asyncpg://minetracker_user:minetracker_password@localhost:5433/minetracker

```



### `REDIS_URL`



**Назначение**: URL-адрес для подключения к Redis (кэширование, Rate Limit). Если не задан, бот попытается подключиться к локальному Redis.



**Пример**:

```env

REDIS_URL=redis://localhost:6379/0

```

## Локальный запуск (Разработка)



Для локального запуска рекомендуется поднять инфраструктуру (PostgreSQL + Redis) через Docker, а сам бот запускать локально.



### Шаг 1: Поднятие инфраструктуры



В файле `docker-compose.yml` уже прописаны сервисы базы данных и Redis. Вы можете поднять только их:



```bash

docker-compose up -d postgres redis

```



База данных будет доступна на порту `5433` (согласно настройкам компоуза), а Redis на `6379`.



### Шаг 2: Настройка .env



Создайте `.env` файл и укажите данные для подключения:



```env

BOT_TOKEN=ВАШ_ТОКЕН

DB_DSN=postgresql+asyncpg://minetracker_user:minetracker_password@localhost:5433/minetracker

REDIS_URL=redis://localhost:6379/0

```



### Шаг 3: Установка зависимостей



В проекте используется `uv`. Выполните:



```bash

uv sync

```



Или стандартным способом:

```bash

python -m venv .venv

source .venv/bin/activate

pip install -r requirements.txt

```



### Шаг 4: Накат миграций



Перед запуском инициализируйте базу данных (создание таблиц):



```bash
# Если использовали uv:
uv run alembic upgrade head

# Если используете стандартный venv (с активированным окружением):
alembic upgrade head
```



### Шаг 5: Запуск бота



```bash
# Если использовали uv:
uv run python -m mc_ping_bot

# Если используете стандартный venv (с активированным окружением):
python -m mc_ping_bot
```

## Запуск через Docker Compose (Production)



Если вы хотите запустить весь проект, включая самого бота, в Docker-контейнерах:



1. Убедитесь, что ваш `.env` файл настроен правильно (в случае Compose бот обращается к сервисам по их именам):



```env

BOT_TOKEN=ВАШ_ТОКЕН

DB_DSN=postgresql+asyncpg://minetracker_user:minetracker_password@postgres:5432/minetracker

REDIS_URL=redis://redis:6379/0

```



2. Соберите и запустите контейнеры:



```bash

docker-compose up -d --build

```



3. (Важно) Накатите миграции БД внутри контейнера бота:



```bash

docker exec -it minetracker_bot alembic upgrade head

```



Бот будет автоматически перезапускаться при падениях благодаря флагу `restart: always`. Вы можете просматривать логи через `docker logs -f minetracker_bot`.
