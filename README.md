# MineTracker



Проект Telegram-бота для мониторинга серверов Minecraft, написанный на `Python 3.13` с использованием `Aiogram 3` и базы данных `PostgreSQL`.



Бот совмещает в себе:



- публичную часть для отслеживания онлайна серверов Minecraft для пользователей;

- глобальный кэш через Redis для предотвращения Rate-limit со стороны Telegram API;

- панель администратора/модератора для управления тикетами и серверами;

- систему ролей пользователей (юзер, модератор, администратор);

- премиум-подписки и ограничения по лимитам серверов.



README описывает фактическое состояние репозитория на текущий момент, включая реализованные возможности и известные ограничения.

## Назначение проекта



Проект предназначен для автоматизированного мониторинга серверов Minecraft и включает несколько сценариев использования:



- проверка пинга, MOTD, TPS и текущего онлайна;

- добавление серверов на авто-мониторинг в личные сообщения или группы;

- интеграция в игровые чаты и темы (Threads / Topics) Telegram-супергрупп;

- административное управление тикетами и модерация серверов.



## Текущий статус



Проект активно развивается. Ниже — честная картина по состоянию на сейчас.



### ✅ Полностью реализовано



**Публичный функционал бота:**

- Получение информации о сервере по IP: пинг, игроки, версия, MOTD (через библиотеку `mcstatus`).

- Привязка серверов к чатам (ЛС, группы, каналы) с автообновлением статуса через фоновые воркеры.

- Настройки отображения мониторинга: возможность скрыть TPS, MOTD, список игроков.

- Индивидуальные меню команд для различных пользователей (в зависимости от роли) с кешированием в Redis во избежание флуда API Telegram.



**Административная панель и премиум:**

- Ролевая модель: разделение команд для обычных пользователей, модераторов и администраторов.

- Фоновый воркер для даунгрейда и отключения истекших премиум-подписок.

- Система тикетов и панель модератора.



**Инфраструктура:**

- Внедрение зависимостей (Dependency Injection) для сессии БД через Middleware `DatabaseMiddleware`.

- Защита от флуда и спама через `RateLimitMiddleware` (на базе Redis).

- Alembic для накатывания и миграции схемы БД.

- Docker и Docker Compose для развертывания проекта "одной кнопкой".



## Технологический стек



- `Python 3.13`

- `Aiogram 3` (3.24.0) — асинхронный фреймворк для Telegram Bot API

- `SQLAlchemy 2` + `asyncpg` — асинхронный ORM и драйвер для работы с БД

- `Alembic` — управление миграциями базы данных

- `Redis` (`redis.asyncio`) — хранение сессий, кэша и ограничений (Rate Limit)

- `mcstatus` (12.1.0) — ядро для проверки состояния Minecraft серверов

- `Pydantic Settings` — валидация конфигурации и чтение `.env`

- `structlog` / `logging` — логирование



## Требования к окружению



- `Node.js` не используется, стек полностью на Python.

- `Python`: рекомендуется актуальная `3.13` версия (или `uv` менеджер пакетов, судя по `uv.lock`).

- `PostgreSQL` версии 15+ для базы данных.

- `Redis` версии 7+ для кэширования.



## Установка и запуск



### 1. Установка зависимостей



Рекомендуется использовать менеджер `uv` или классический `pip` с виртуальным окружением:



```bash

uv sync

# или если используется pip

python -m venv .venv

source .venv/bin/activate

pip install -r requirements.txt # или через pyproject.toml

```



### 2. Настройка окружения



Создайте файл `.env` в корне проекта (вы можете использовать `config.yaml.example` и `.env.example` как ориентиры):



```env

BOT_TOKEN=YOUR_BOT_TOKEN_HERE

DB_DSN=postgresql+asyncpg://minetracker_user:minetracker_password@localhost:5432/minetracker

REDIS_URL=redis://localhost:6379/0

```



### 3. Применение миграций



Перед первым запуском необходимо создать таблицы в базе данных:



```bash

alembic upgrade head

```



### 4. Запуск бота



```bash

python -m mc_ping_bot

```



## Docker и развертывание



Репозиторий содержит готовый `docker-compose.yml` и `Dockerfile` для быстрого развертывания.



В Compose-файл включены:

- `postgres:15-alpine` (СУБД)

- `redis:7.2-alpine` (Кэш)

- `bot` (Сам Telegram бот)



Запуск всего комплекса:



```bash

docker-compose up -d --build

```



## Переменные окружения



### `BOT_TOKEN`



Токен вашего Telegram-бота, полученный у @BotFather.



### `DB_DSN`



Асинхронный DSN для подключения к PostgreSQL через `asyncpg`. Используется в `config.py` и `alembic.ini`.



Пример: `postgresql+asyncpg://user:pass@host:port/dbname`



### `REDIS_URL`



URL-адрес подключения к Redis для кэширования команд и Rate Limit.



Пример: `redis://localhost:6379/0`



## Команды и функциональность



У бота настроено динамическое распределение команд через `update_user_commands()` с использованием Redis для кэширования.



### Пользовательские команды (ЛС)



| Команда | Описание |

| --- | --- |

| `/start` | Запустить бота |

| `/info` | Статус сервера |

| `/players` | Список игроков онлайн |

| `/ping` | Проверить пинг до сервера |

| `/motd` | Получить описание (MOTD) сервера |

| `/version` | Узнать версию Minecraft сервера |

| `/ip` | Получить IP адрес (резолв) |

| `/help` | Справка |

| `/commands` | Обновить меню команд (если изменилась роль) |

| `/configuration` | Настройки мониторинга |

| `/set_chat_server` | Привязать сервер для мониторинга в текущем чате |

| `/set_chat_gamechat` | Настроить игровой чат (Threads / Topics) |



### Административные команды



Доступны только для пользователей с ролями `admin` и `moder`:



| Команда | Описание |

| --- | --- |

| `/moder` | Открыть панель модератора |

| `/tickets` | Управление тикетами поддержки |



## Структура базы данных



БД состоит из нескольких ключевых таблиц (определены в `mc_ping_bot/db/models.py`):



- `users` — профили пользователей, их Telegram ID, язык (`lang`), роль и премиум-статус.

- `chats` — сущность чатов, групп и каналов (`private`, `group`, `supergroup`, `channel`), куда добавлен бот.

- `servers` — уникальные связки IP + порт серверов.

- `monitored_servers` — связующая таблица (M:N) между `chats` и `servers`. Хранит настройки отображения (`show_tps`, `show_motd`), ID закрепленного сообщения с автообновлением и интервалы.

## Разработка и вклад в проект



Правила командной работы и стандарты оформления кода описаны в `CONTRIBUTING.md`. Ключевые моменты:



- используйте `uv` для работы с зависимостями;

- для создания миграций при изменении `models.py` используйте: `alembic revision --autogenerate -m "comment"`;

- используйте `AsyncSession` через Dependency Injection во всех хендлерах;

- не вносите логику проверки серверов в хендлеры, выносите ее в `mc_ping_bot/services/`.
